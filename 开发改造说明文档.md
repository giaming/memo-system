## 3. 新增模块清单

### 3.1 优先级枚举类

**文件路径**: `src/main/java/com/gzeic/memosystem/enums/PriorityEnum.java`

**功能说明**:
- 定义优先级枚举：HIGH（高）、MEDIUM（中）、LOW（低）
- 提供 fromValue() 和 fromName() 方法用于类型转换
- 默认值为 MEDIUM

**枚举值定义**:
```java
HIGH("高优先级", 1),
MEDIUM("中优先级", 2),
LOW("低优先级", 3)
```

### 3.2 分页查询请求DTO

**文件路径**: `src/main/java/com/gzeic/memosystem/dto/MemoPageRequest.java`

**功能说明**:
- 支持分页参数（pageNum, pageSize）
- 支持多条件筛选（title, priority, completed, dueDate范围）
- 支持自定义排序（orderBy, sort）

**请求参数**:
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| pageNum | Integer | 否 | 页码，默认1 |
| pageSize | Integer | 否 | 每页条数，默认10，最大100 |
| title | String | 否 | 标题关键字（模糊查询） |
| priority | String | 否 | 优先级（HIGH/MEDIUM/LOW） |
| completed | Boolean | 否 | 完成状态 |
| startDueDate | LocalDate | 否 | 截止日期范围开始 |
| endDueDate | LocalDate | 否 | 截止日期范围结束 |
| orderBy | String | 否 | 排序字段 |
| sort | String | 否 | 排序方向（asc/desc） |

### 3.3 每日待办汇总响应DTO

**文件路径**: `src/main/java/com/gzeic/memosystem/dto/DailySummaryResponse.java`

**功能说明**:
- 返回每日待办汇总统计数据

**响应字段**:
| 字段 | 类型 | 说明 |
|------|------|------|
| todayTotal | Integer | 今日待办总数 |
| completed | Integer | 已完成数 |
| uncompleted | Integer | 未完成数 |
| highPriority | Integer | 高优先级数 |
| overdue | Integer | 逾期数 |
| summaryDate | String | 统计日期 |

## 4. 修改模块清单

### 4.1 MemoRequest DTO

**文件路径**: `src/main/java/com/gzeic/memosystem/dto/MemoRequest.java`

**修改内容**:
- `priority` 字段从 Integer 改为 String（支持枚举名称）
- `dueDate` 字段从 LocalDateTime 改为 LocalDate
- 添加 `getPriorityEnum()` 和 `getPriorityValue()` 方法

### 4.2 MemoService 接口

**文件路径**: `src/main/java/com/gzeic/memosystem/service/MemoService.java`

**新增方法**:
- `IPage<Memo> pageMemos(MemoPageRequest request, Long userId)` - 分页查询
- `Memo setCompleted(Long id, Boolean completed, Long userId)` - 设置完成状态
- `DailySummaryResponse getDailySummary(Long userId)` - 每日汇总

### 4.3 MemoServiceImpl 实现

**文件路径**: `src/main/java/com/gzeic/memosystem/service/impl/MemoServiceImpl.java`

**修改内容**:
- 更新 createMemo() 和 updateMemo() 方法支持优先级枚举
- 实现 pageMemos() 方法支持多条件筛选和分页
- 实现 setCompleted() 方法支持设置完成状态
- 实现 getDailySummary() 方法返回每日汇总数据
- 添加 sendMemoCreatedNotify() 和 sendMemoUpdatedNotify() 方法发送异步通知

### 4.4 MemoController 控制器

**文件路径**: `src/main/java/com/gzeic/memosystem/controller/MemoController.java`

**新增接口**:
- `GET /api/memo/page` - 分页查询备忘录
- `GET /api/memo/daily-summary` - 每日待办汇总
- `PATCH /api/memo/{id}/status?completed=true/false` - 设置完成状态

**修改接口**:
- 状态切换接口路径从 `PUT /api/memo/{id}/toggle` 保留为兼容接口
- 新增 `PATCH /api/memo/{id}/status` 作为标准接口

### 4.5 UserServiceImpl 实现

**文件路径**: `src/main/java/com/gzeic/memosystem/service/impl/UserServiceImpl.java`

**修改内容**:
- `login()` 方法返回格式调整为 `{token, userId, username}`
- `register()` 方法添加确认密码校验
- 用户名统一转为小写存储和查询

### 4.6 MemoDueMessage 消息体

**文件路径**: `src/main/java/com/gzeic/memosystem/mq/MemoDueMessage.java`

**新增字段**:
- `actionType` - 操作类型（created/updated/due_reminder）
- `timestamp` - 消息发送时间

### 4.7 前端页面 memos.html

**文件路径**: `src/main/resources/templates/memos.html`

**新增功能**:
- 每日待办汇总卡片展示
- 搜索框（标题、优先级、状态筛选）
- 分页控件
- 优先级标签显示
- 截止日期和逾期提醒显示
- 使用新版 PATCH 接口更新完成状态

## 7. 数据库变更

### 7.1 数据库表结构（V1.1）

已按照需求规格说明书更新数据库表结构，与Entity类完全对应。

#### 用户表 (user)

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | BIGINT | PRIMARY KEY, AUTO_INCREMENT | 用户主键 |
| tenant_id | BIGINT | NOT NULL, INDEX | 租户ID（多租户隔离） |
| username | VARCHAR(50) | NOT NULL | 用户名 |
| password | VARCHAR(255) | NOT NULL | BCrypt加密后的密码 |
| email | VARCHAR(100) | | 邮箱（可选） |
| phone | VARCHAR(20) | | 手机号（可选） |
| create_time | DATETIME | NOT NULL | 创建时间（自动填充） |
| update_time | DATETIME | NOT NULL | 更新时间（自动更新） |

#### 备忘录表 (memo)

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | BIGINT | PRIMARY KEY, AUTO_INCREMENT | 备忘录主键 |
| tenant_id | BIGINT | NOT NULL, INDEX | 租户ID（多租户隔离） |
| title | VARCHAR(100) | NOT NULL | 标题（需求：最大100字符） |
| content | VARCHAR(500) | | 内容（需求：最大500字符） |
| user_id | BIGINT | NOT NULL, INDEX | 所属用户ID |
| is_completed | TINYINT(1) | NOT NULL DEFAULT 0 | 是否已完成（0/1） |
| is_notified | TINYINT(1) | NOT NULL DEFAULT 0 | 是否已发送提醒（幂等控制） |
| priority | INT | NOT NULL DEFAULT 2 | 优先级（1=HIGH, 2=MEDIUM, 3=LOW） |
| due_date | DATETIME | | 截止日期（可选） |
| create_time | DATETIME | NOT NULL | 创建时间（自动填充） |
| update_time | DATETIME | NOT NULL | 更新时间（自动更新） |

#### 提醒记录表 (reminder_log)

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | BIGINT | PRIMARY KEY, AUTO_INCREMENT | 主键 |
| memo_id | BIGINT | NOT NULL, INDEX | 备忘录ID |
| user_id | BIGINT | NOT NULL, INDEX | 用户ID |
| tenant_id | BIGINT | NOT NULL, INDEX | 租户ID |
| remind_time | DATETIME | NOT NULL | 提醒时间 |
| remind_type | VARCHAR(20) | | 提醒类型（DUE/EARLY） |

### 7.2 数据库初始化

执行以下SQL文件初始化数据库：
- `sql/init.sql` - 完整初始化脚本
- `src/main/resources/db.sql` - 应用资源文件

```bash
mysql -u root -p < sql/init.sql
```

## 8. 部署说明

### 8.1 环境要求

- JDK 17 或更高版本（推荐 JDK 21）
- Maven 3.6+
- MySQL 8.0+
- Redis 7.x
- RabbitMQ 3.x

### 8.2 构建命令

```bash
# 清理并编译
mvn clean compile

# 运行测试
mvn test

# 打包
mvn clean package

# 运行
java -jar target/memo-system-0.0.1-SNAPSHOT.jar
```

### 8.3 Docker 部署

使用项目根目录下的 `docker-compose.yml` 进行一键部署：

```bash
docker-compose up -d
```
